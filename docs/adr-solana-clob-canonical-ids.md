# ADR: Solana CLOB Canonical Program IDs

**Status:** Accepted  
**Date:** 2026-02-25  
**Author:** Agent A (automated)

## Context

The Solana program deployment suffered from program ID drift across four layers:
1. `declare_id!()` in Anchor program source
2. `Anchor.toml` cluster sections
3. App IDL JSON files (`app/src/idl/*.json`)
4. Hardcoded fallback IDs in `programs.ts` and `market-maker-bot`

Each layer had different program IDs, causing runtime failures when the app
tried to interact with on-chain programs using stale addresses.

## Decision

We designate the following as the **canonical program IDs** for all clusters
(devnet, localnet, mainnet) until redeployment with new keypairs:

| Program | Canonical Program ID |
|---------|---------------------|
| `gold_clob_market` | `AqRu5b1fd67VyR4MgjKPN9EMgQ8wxauDUxyY5pUsGdAW` |
| `fight_oracle` | `4fvVdiZkMQQGjYWHyKubjkdhh1DfJaYNvaRvRWPeKcMN` |
| `gold_perps_market` | `3WKQf3J4B8QqRyWcBLR7xrb9VFPVjkZwzyZS67AahDbK` |

### Sources of Truth (priority order)

1. `declare_id!()` in `programs/*/src/lib.rs` — ultimate source
2. `Anchor.toml` `[programs.*]` sections — must match declare_id
3. `anchor/target/idl/*.json` — generated by `anchor build`
4. `app/src/idl/*.json` — copied from target by sync script

### Sync Mechanism

- **Script:** `scripts/sync-anchor-idl-to-app.mjs` copies IDLs from anchor target to app
- **CI Guard:** `.github/workflows/idl-sync-check.yml` runs `--check` mode on PRs
- **Workflow:** After any `anchor build`, run the sync script before committing

## Consequences

- All runtime program resolution now points to a single canonical ID per program
- Fallback IDs in `programs.ts` and `market-maker-bot/src/index.ts` match canonical IDs
- CI will block PRs that introduce IDL drift
- Future redeployments must update `declare_id!()` first, then `anchor build`, then sync
